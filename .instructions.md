# Backend Integration Instructions - Strictly Enforce

## Critical API Contract Requirements

### 1. ✅ ALWAYS Valid JSON Responses
- **Every** API response MUST be valid JSON
- **Every** endpoint MUST return `{ success: boolean, message: string, data: any, error?: any }`
- If error occurs, MUST return `{ success: false, error: { message: "text" }, ... }`
- **NO** empty responses, **NO** plain text, **NO** null without wrapper
- Test: `JSON.parse(response)` must NOT throw

### 2. 📋 Status Enums - UPPERCASE ONLY
- **Students**: `ACTIVE | INACTIVE | GRADUATED`
- **Payments**: `PAID | UNPAID | PARTIAL | OVERDUE`
- **Attendance**: `present | absent | late | excused`
- Response data MUST use these exact values (case-sensitive)
- NO lowercase variations

### 3. 📚 List Responses - Array or Wrapped Object
All list endpoints MUST return one of:
```json
// Direct array
[{ id: "...", name: "..." }]

// or Wrapped array (any of these keys)
{ "groups": [...], "data": [...], "items": [...], "rows": [...], "students": [...], "payments": [...], "entries": [...], "attendance": [...] }
```

### 4. 👨‍🎓 Nested Student Objects
When student is nested in attendance/payment response:
```json
{
  "id": "unique-id",
  "student": {
    "id": "student-id",
    "firstName": "Abdulla",
    "lastName": "Karimov",
    "fullName": "Abdulla Karimov"
  },
  "studentId": "student-id",      // Fallback if no nested student
  "studentName": "Abdulla Karimov" // Fallback if no nested student
}
```

### 5. 🎯 Endpoint Specifications

#### GET /api/groups
- Returns: `{ success: true, data: [group, ...] }` or `{ success: true, data: { groups: [...] } }`
- **Fix**: Ensure groups array is valid, NOT empty, includes all groups (Nodirjon Group MUST be present)
- Required fields: `id, name, subject, teacherId, monthlyFee, isActive`

#### GET /api/students
- Returns: array of students (112+ expected)
- **Fix**: Must return students array, not empty
- Required fields: `id, firstName, lastName, fullName, phone, status, groups`
- Status must be UPPERCASE (ACTIVE/INACTIVE/GRADUATED)

#### POST /api/lessons
- Input: `{ groupId: string, lessonDate: string (YYYY-MM-DD) }`
- Returns: `{ success: true, data: { id: "lesson-id", ... } }`
- **Fix**: MUST return object with `id` field in `data`, NOT in root
- Response example: `{ success: true, message: "Lesson created", data: { id: "123abc", groupId: "...", lessonDate: "2026-03-28" } }`

#### GET /api/attendance
- Returns: `{ success: true, data: [attendance-record, ...] }` or `{ success: true, data: { entries: [...] } }`
- If `lessonId` param: may return bare array OR wrapped in object
- **Fix**: MUST return valid JSON array of attendance records, NEVER null/empty string
- Required fields: `id, studentId, studentName, status`
- Optional but preferred: `student: { id, firstName, lastName, fullName }`

#### GET /api/payments
- Returns: `{ success: true, data: [payment, ...] }` or `{ success: true, data: { payments: [...] } }`
- Query params: `?groupId=...&billingMonth=...&status=...`
- **Fix**: Wrap response in data object, handle all query filters
- Status MUST be UPPERCASE (PAID/UNPAID/PARTIAL/OVERDUE)

#### GET /api/payments/summary
- **REQUIRED Query**: `groupId` (MANDATORY, no null/empty)
- Input: ` { groupId: string, billingMonth: string }`
- Returns:
```json
{
  "success": true,
  "data": {
    "totals": {
      "expectedAmount": 500000,      // PREFERRED
      "collectedAmount": 300000,     // PREFERRED
      "debtAmount": 200000,
      "totalAmount": 500000,         // FALLBACK
      "paidAmount": 300000           // FALLBACK
    },
    "paidStudents": [... student array with amounts ...],
    "debtors": [... student array with debt amounts ...]
  }
}
```
- **Fix**: groupId parameter MUST NOT be optional; summary only valid per group

### 6. 🔐 Error Handling
- All errors MUST return:
  ```json
  {
    "success": false,
    "message": "Human-readable error message in Uzbek",
    "error": {
      "message": "Detailed error message",
      "code": "ERROR_CODE",
      "fieldErrors": { "fieldName": "Error message", ... }
    }
  }
  ```
- **Fix**: Check lesson/attendance endpoints for proper error responses (not throwing 500 without JSON wrapper)
